# Grading Rubric

Applies to every lab in this repository, per the course-wide standard
(Phase 1 Instructor Pack, Section 5.2).

## Lab scoring (every lab)

| Component | Weight | What "full marks" looks like |
|---|---|---|
| Feature works | 70% | All acceptance criteria pass, endpoints respond correctly |
| Unit tests | 20% | Minimum test count met, all tests pass (`mvn test` = green) |
| Code quality | 10% | Clean code, no obvious duplication, meaningful names |

- **Partial credit on Feature:** pro-rated by how many acceptance criteria pass.
- **Zero on Unit Tests:** no tests at all is the single most common deduction —
  don't let this be you.
- **Code Quality** is reviewed during lab debrief; it's subjective but applied
  consistently across all trainees.
- Bonus points are marked explicitly in each lab doc (`docs/labs/`). Bonus
  points can raise an individual lab score but never the overall course
  grade above 100%.
